# Iterative Magnitude Pruning on Mamba SSM

## Goal

Determine the **sparsity tolerance** of a Mamba-based State Space Model (SSM)
through Iterative Magnitude Pruning (IMP), applied to a chaotic time-series
forecasting task (Mackey-Glass equations).  We seek the *breaking point* - the
maximum sparsity level before the model's continuous-time dynamics lose
predictive power.

---

## Methodology

### Dataset
The **Mackey-Glass delay-differential equation** generates a normalised,
mildly/strongly chaotic 1-D time series:

```
dx/dt = beta·x(t-tau) / (1 + x(t-tau)^n) - gamma·x(t)
tau=17 -> mild chaos   |   tau=30 -> strong chaos
```

The series (12 000 steps) is split 70 / 15 / 15 % into train / val / test.
Sliding windows of length 128 are used to predict the next single step.

A **corrupted test set** is also generated by injecting:
- Random missing values (5 % of points, replaced by linear interpolation)
- Additive signed spikes (3 % of points, +-3std)

### Model
`MambaForecaster`: stacked Mamba blocks (from `mamba-ssm`) between a linear
input projection and a linear forecast head.

```
input  (B, L, 1)
  |
  Linear(1 -> d_model)
  |
  [MambaBlock + residual] x N
  |
  RMSNorm
  |
  Linear(d_model -> horizon)
  |
output (B, horizon)
```

Default: `d_model=64, d_state=16, d_conv=4, expand=2, n_layers=4`.

### Pruning
**Global unstructured magnitude pruning** (IMP):

1. Train a dense baseline (30 epochs, Adam + cosine LR, early stopping).
2. Save the original dense weights (lottery-ticket style).
3. For each target sparsity level in a log-spaced schedule `[0%, …, 99%]`:
   - Compute a global magnitude threshold on the **original** weights.
   - Zero out all weights below the threshold (apply binary masks).
   - Register gradient hooks to keep pruned weights at zero.
   - Fine-tune for 5 epochs at a lower LR (3e-4).
   - Record val / test MSE and anomaly-test MSE.

Three parameter groups are **intentionally excluded** from pruning:
- `A_log` - SSM eigenvalue parameter; zeroing it collapses the recurrent dynamics entirely
- `D` - scalar skip connection (one float per channel); too few parameters to matter
- all biases - standard IMP convention; biases are small in count and critical for activation shifts

### Breaking point criterion
`val_MSE > max(5 x baseline_val_MSE, 1e-3)`

The floor `1e-3` prevents false triggers when `baseline_val_MSE` is near zero (near-perfect dense fit), where even numerical noise would satisfy the relative criterion alone.

### Confidence intervals
The experiment runs with **10 random seeds**; plots show mean +- 1std bands.

## Limitations

- Only univariate forecasting (`horizon=1`). Multi-step prediction may be more sensitive to sparsity.
- Magnitude pruning is **unstructured** - structured pruning (e.g. removing entire state dimensions) would likely break dynamics at lower sparsity.
- Fine-tuning budget is intentionally small (5 epochs) to reflect a realistic post-deployment scenario; a larger budget could partially recover accuracy.
- Results are specific to Mamba v2 (`mamba-ssm==2.2.2`); other SSM variants (S4, S6, RWKV) may behave differently.
- Breaking point detection is heuristic (`val_MSE > max(5x baseline, 1e-3)`); a smoother metric (e.g. AUC of the sparsity curve) could be more principled.

## How to run

### Google Colab (recommended - requires T4 GPU)

1. Open `notebook.ipynb` in Colab.
2. Set runtime: **Runtime -> Change runtime type -> T4 GPU**.
3. Run all cells in order. Cell 0 installs dependencies, cell 1 clones the repo.

### Locally - CPU-only tests (Linux / WSL / Windows)

`test_dataset.py` and `test_pruning.py` do not need CUDA or `mamba-ssm`:

```bash
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
pip install torch --index-url https://download.pytorch.org/whl/cpu
pytest tests/test_dataset.py tests/test_pruning.py -v
```

### Locally - full experiment (Linux / WSL with CUDA GPU)

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
pip install torch==2.4.0 --index-url https://download.pytorch.org/whl/cu121
pip install causal-conv1d==1.4.0
pip install mamba-ssm==2.2.2 --no-build-isolation
pytest tests/ -v
```

And to run the full IMP experiment:

```python
from magnitude_pruning.experiment import ExperimentConfig, run_experiment
import numpy as np

cfg = ExperimentConfig(device="cuda")  # defaults: 3 seeds, tau=17, 14 sparsity steps
results = run_experiment(cfg, verbose=True)

bps = [b for b in results["breaking_points"] if b is not None]
print(f"Breaking point: {np.median(bps)*100:.1f}% +- {np.std(bps)*100:.1f}%")
```
