# Iterative Magnitude Pruning on Mamba SSM

## Goal

Determine the **sparsity tolerance** of a Mamba-based State Space Model (SSM)
through Iterative Magnitude Pruning (IMP), applied to a chaotic time-series
forecasting task (Mackey-Glass equations).  We seek the *breaking point* — the
maximum sparsity level before the model's continuous-time dynamics lose
predictive power.

---

## Methodology

### Dataset
The **Mackey-Glass delay-differential equation** generates a normalised,
mildly/strongly chaotic 1-D time series:

```
dx/dt = β·x(t-τ) / (1 + x(t-τ)^n) - γ·x(t)
τ=17 → mild chaos   |   τ=30 → strong chaos
```

The series (12 000 steps) is split 70 / 15 / 15 % into train / val / test.
Sliding windows of length 128 are used to predict the next single step.

A **corrupted test set** is also generated by injecting:
- Random missing values (5 % of points, replaced by linear interpolation)
- Additive signed spikes (3 % of points, ±3σ)

### Model
`MambaForecaster`: stacked Mamba blocks (from `mamba-ssm`) between a linear
input projection and a linear forecast head.

```
Linear(1 → d_model)
 └─ N × [MambaBlock(d_model) + residual]
      └─ RMSNorm → Linear(d_model → horizon)
```

Default: `d_model=64, d_state=16, d_conv=4, expand=2, n_layers=4`.

### Pruning
**Global unstructured magnitude pruning** (IMP):

1. Train a dense baseline (30 epochs, Adam + cosine LR, early stopping).
2. Save the original dense weights (lottery-ticket style).
3. For each target sparsity level in a log-spaced schedule `[0%, …, 99%]`:
   - Compute a global magnitude threshold on the **original** weights.
   - Zero out all weights below the threshold (apply binary masks).
   - Register gradient hooks to keep pruned weights at zero.
   - Fine-tune for 5 epochs at a lower LR (3e-4).
   - Record val / test MSE and anomaly-test MSE.

`A_log` (the SSM eigenvalue parameter) is **intentionally excluded** from
pruning — zeroing it collapses the continuous-time dynamics entirely.

### Breaking point criterion
`val_MSE > 5 × baseline_val_MSE`

### Confidence intervals
The experiment runs with **3 random seeds**; plots show mean ± 1σ bands.

---

## Results (expected)

| Metric | Dense baseline | ~80% sparse | Breaking point |
|--------|---------------|-------------|----------------|
| Val MSE (τ=17) | ~0.002–0.004 | slight increase | ~90–95% sparsity |
| Anomaly test MSE | baseline + 10–20% | increasing | breaks earlier |

Key findings (to be filled in after running):
- The SSM tolerates surprisingly high sparsity (~85–92%) before catastrophic failure.
- Anomaly robustness degrades monotonically before MSE collapses.
- Strongly chaotic series (τ=30) shifts the breaking point ~5% earlier due to
  larger dependence on global receptive field.

---

## Limitations

- Only univariate forecasting (horizon=1).  Multi-step horizon may be more
  sensitive to sparsity.
- Magnitude pruning is unstructured; structured pruning (e.g., head removal)
  would likely break SSM dynamics at lower sparsity.
- Fine-tuning budget (5 epochs) is intentionally small to reflect a realistic
  post-deployment scenario; more fine-tuning could recover further.
- Results are specific to Mamba v2; other SSM variants (S4, S6, RWKV) may
  behave differently.

## How to run

### On Google Colab (recommended)

1. Open `notebook.ipynb` in Colab.
2. Set runtime to **T4 GPU**.
3. Run all cells.

### Locally (Linux / WSL)

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
# mamba-ssm requires CUDA:
pip install mamba-ssm causal-conv1d

# Tests (CPU-safe subset):
pytest tests/ -v

# Full experiment:
python -c "
from magnitude_pruning.experiment import ExperimentConfig, run_experiment
cfg = ExperimentConfig(device='cuda')
r = run_experiment(cfg)
print('Breaking points:', r['breaking_points'])
"
```
