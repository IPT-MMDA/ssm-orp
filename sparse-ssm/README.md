# Технічний звіт: Реплікація методу SparseSSM для Mamba-130M

## 1. Об'єкт дослідження
Проведено незалежне тестування алгоритму **Time-Selective Mask Aggregation** (Tuo & Wang, 2025) для прунінгу параметрів $A_{\log}$ у архітектурі Mamba. Дослідження фокусувалося на перевірці ефективності OBS-апроксимації (Optimal Brain Surgeon) для неструктурного та структурного стиснення SSM-модулів.

## 2. Порівняльний аналіз результатів
Експерименти на валідаційній вибірці **WikiText-2** демонструють перевагу запропонованого методу над стандартними підходами:

| Конфігурація | Перплексія (PPL) | Деградація (%) | Примітка |
| :--- | :--- | :--- | :--- |
| **Dense Baseline** | 22.82 ± 0.7 | — | HF `MambaForCausalLM` |
| **SSM Unstructured (50%)** | **25.48 ± 0.8** | **+11.7%** | Algorithm 1 (OBS) |
| **SSM Structured (50%)** | 30.29 ± 0.9 | +32.7% | Видалення стовпців $N$ |
| **Magnitude Baseline** | ~740 | >3000% | Наївне обнулення |

## 3. Ключові технічні висновки

### 3.1. Динаміка обнулення параметрів
У процесі реалізації підтверджено, що встановлення $A_{\log} = 0$ не призводить до вимкнення каналу, а перетворює його на інерційний інтегратор ($\Delta A \approx 0.97$). Для коректної нейтралізації обрізаних елементів використано граничне значення $A_{\log} = +38$ (max safe fp16), що забезпечує асимптотичне наближення $\Delta A \to 0$.

### 3.2. Роль часової агрегації (Algorithm 1)
Ефективність методу базується на трифазному підході. Накопичення статистики активацій прихованого стану $h_{t-1}$ дозволяє виявити часову динаміку, яку ігнорує статичний L2-аналіз. Аналіз лічильників кандидатів (`C_count`) підтверджує значну диференціацію важливості елементів: частина параметрів є критичними на кожному кроці послідовності, тоді як інші демонструють високу надлишковість.

### 3.3. Обмеження напівструктурного прунінгу (N:M)
Спроби впровадження режиму 2:4 для Mamba-130M призвели до критичного зростання PPL. Встановлено, що моделі малого масштабу не мають достатньої параметричної надлишковості для задоволення жорстких групових обмежень у межах матриць $[768, 16]$.



## 4. Оптимізація обчислювальної фази
Для скорочення часу калібрації впроваджено такі рішення:
* **Векторизація SSM scan:** Попереднє обчислення дискретних параметрів $\Delta A, \Delta B$ для всієї послідовності до входу в основний цикл.
* **Batched Top-K:** Заміна ітеративного пошуку порогів на кожному кроці $t$ єдиним викликом `topk` з наступною агрегацією через `bincount`. Це дозволило прискорити обробку чанка в **3.7 раза**.

## 5. Висновок
Реплікація підтверджує життєздатність OBS-підходу для селективних моделей простору станів. Хоча прунінг матриць $A_{\log}$ зачіпає лише **0.46%** загальних параметрів моделі, він є критично важливим етапом для збереження функціональної динаміки перед подальшим стисненням важких FFN-компонентів (in/out_proj).
